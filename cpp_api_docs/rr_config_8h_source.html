18 /**
19  * read or store default values.
20  *
21  * Many of RoadRunner classes use a number of configration parameters. Most of these can be set
22  * using the Config class. The values stored in the Config class only determine the defaut values of
23  * parameters.
24  *
25  * The Config class will look in the following locations for the config file, and will load the
26  * values from the first config file it finds. If it does not find a config file in one of the
27  * following locations, a default set of configuration parameters are used. The search locations of
28  * the config file are:
29  *
30  * #1: the ROADRUNNER_CONFIG environment variable
31  *
32  * #2: try the user's home directory for roadrunner.conf, i.e.:
33  *
34  * /Users/andy/roadrunner.conf
35  *
36  * #3: try the user's home directory for .roadrunner.conf, i.e.:
37  *
38  * /Users/andy/.roadrunner.conf
39  *
40  * #4: try the same directory as the roadrunner shared library, this
41  * will be the same directory as the python _roadrunner.pyd python extension module, i.e.:
42  *
43  * /Users/andy/local/lib/roadrunner.conf
44  *
45  * #5: try one directory up from the where the shared library or program is at, i.e.:
46  *
47  * /Users/andy/local/roadrunner.conf
48  *
49  * The conf file is just a plain text file of where each line may
50  * be key / value pair separated by a :", i.e.
51  *
52  * KEY_NAME : Value Any line that does not match this format is ignored, and keys that are not found
53  * are also ignored. Therefore, any line that does not start w* ith a word character is considered a
54  * comment.
55  *
56  * All of the configuration managment functions are static method of the Config class, and all of
57  * the configuration keys are static attributes * of the Config class, these are documented in the
58  * Configuration Functions section.
59  *
60  * As all of the Config class methods are static, one never instantiates the Config class.
61  */
