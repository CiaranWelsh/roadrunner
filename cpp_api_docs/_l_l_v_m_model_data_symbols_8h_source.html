93 /**
94  * stores the names of all the symbols in the sbml model and thier
95  * indexes in the ModelData arrays. This class essentially maps symbol
96  * names into ModelData indexes.
97  *
98  * The logic behind splitting up LLVMModelDataSymbols and LLVMModelSymbols
99  * is that LLVMModelSymbols contains all the ASTs from the model, and this
100  * contains only ModelData indices -- once the all the model functions
101  * are generated, there is no need to keep AST information around, but we
102  * still need to know what symbol is at what index for the various
103  * Model accessor functions.
104  *
105  * * data storage:
106  * There can exist rules (assigment, rate, and eventually algebraic) rules
107  * that determine the value of a symbol.
108  *
109  * All items for which a rate rule exists are stored in the
110  * modelData::rateRules array.
111  *
112  * No space is allocated for items determined by rate rules.
113  *
114  * * All elements get an index value, even the dependent ones, this allows
115  * us uniquely indentify them in the generated accessor functions.
116  *
117  * * Most of the indexes used in this class are indexes into ModelData
118  * arrays, therefore we use unsigned integers -- these are less error
119  * prone and its easier to check if they are valid i.e. only check that
120  * they are less than the the array size and we do not have to check that
121  * it is positive.
122  *
123  * * All symbols from the sbml are reordered such that the independent
124  * ones are first, followed by the dependent values.
125  */
